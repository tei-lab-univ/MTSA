# CONTRIBUTING

MTSA - Tei Lab Edition への貢献ありがとうございます。
本プロジェクトでは、本家（GitLab）の更新を取り込みつつ、独自拡張を維持するために以下の運用ルールを設けています。

## 開発フロー

### 1. 本家との同期 (sync)

本家の更新を取り込むための [`Sync from GitLab`](https://github.com/tei-lab-univ/MTSA/actions/workflows/sync.yaml) ワークフローが設定されています。

- **sync ブランチの役割**: このワークフローは自動で本家の最新コードを取得し、 `sync` ブランチを更新します。
- **main への反映**: `sync` ブランチの内容を `main` に取り込む際は、必ず Pull Request (PR) を作成し、マージしてください。
- **直接変更の禁止**: `sync` ブランチはワークフロー（GitHub Actions）専用です。手動での直接プッシュは避けてください。

### 2. コンフリクト時の対応

`sync` ブランチの更新時にコンフリクトが発生した場合は、以下の手順で対応してください。

1. `sync` ブランチから作業用の派生ブランチ（例: `fix-sync-conflicts`）を作成する。
2. ローカルでコンフリクトを解消し、修正をコミットする。
3. そのブランチから `main` に対して PR を作成し、マージを行う。

### 3. main ブランチの保護

- `main` ブランチへの直接プッシュは禁止されています。
- 独自開発・機能追加・同期対応を問わず、すべての変更は必ず **Pull Request** を経由してください。

## 変更時の必須事項

### CHANGELOG の更新

独自機能の追加や修正をマージする際は、必ず `CHANGELOG.md` に変更内容を記載してください。  
※ 本家からの更新（sync）のみをマージする場合は、記載不要です。
